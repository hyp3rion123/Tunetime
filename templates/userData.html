How we process user data

To use TuneTime, users are required to login using their Google and Spotify accounts.

Google: By signing in with Google, users are granting TuneTime permission to view the events listed in their calendars(read-only permissions). TuneTime will use this information to display the user their events in the UI when creating their transitional playlists. The calculation being performed in determining the playlist length is the only way this scope will be used. No event information is retained past the user's interaction with the website other than through request logs which are automatically deleted after 1 week. There is no associated database or user data storing.

Spotify: Signing in with Spotify grants TuneTime the ability to read and modify private playlists on their behalf. The creation of playlists will only occur when the user presses the submit button indicating which songs they wish their playlist to transition between.
